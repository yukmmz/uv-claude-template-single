以下のルールに従って、静的解析と自動修正ループを実行してください。

### Step 0: 実行対象（TARGET）の決定
引数 `{1}` の内容を確認し、実行対象（TARGET）を以下のように決定してください。
- **引数が空、または `{1}` という文字列そのままである場合**: デフォルトの対象として `./src ./scripts ./tests` の3つのディレクトリすべて（サブフォルダ含む）を TARGET とします。
- **具体的なパス（例: `src/nlinfit_flapaero`）が指定されている場合**: そのパスのみを TARGET とします。

### Step 1: Ruff によるフォーマットと自動修正
1. ターミナルで `uv run ruff check --fix TARGET` を実行する。
2. ターミナルで `uv run ruff format TARGET` を実行する。

### Step 2: Mypy による型チェックと反復修正
1. ターミナルで `uv run mypy TARGET` を実行する。
2. もしエラーやワーニングが出た場合、該当ファイルを読み込み、型の不整合を修正する。
3. 修正後、再度 `uv run mypy TARGET` を実行し、**エラーが完全にゼロになるまでこのプロセスを反復**する。

### ⚠️ 厳守事項（Guardrails）
- 型エラーを消すためだけに安易に `Any` や `# type: ignore` を使わないこと。どうしても必要な場合は `# TODO: [ASSUMPTION] 理由` を記載すること。
- コードの元のロジックや振る舞い（SOLID原則）を絶対に破壊しないこと。
- 修正対象は Step 0 で決定した TARGET 以下のファイルに限定すること。
- 全てパスしたら、修正した要点を簡潔に報告すること。
